gantt — the execution timeline (the hero)

KIND = hue · STATUS = treatment theme

The execution Gantt is harmonograf's instrument. It preserves the elegant 2-axis encoding: KIND = hue (9 span kinds, each a terminal ANSI slot from the active theme) and STATUS = treatment (running → the one --accent + breathing; failed → --bad + an ✕; pending → faint; cancelled → hatch; planned → dashed). Status is the only place good/bad/accent appear, so a span is never red for its identity — only when it failed. Below, the same realistic multi-agent trace is drawn two ways.

Option A — fit-to-width SVG line-art (recommended)

A pure viewBox SVG at width:100%: hairline gridlines (--rule-soft, 0.6w), rx:3 bars at reduced fill-opacity lifting to 1 on hover, the agent gutter + lifelines, a context-window band (--caution tint) along the coordinator, TRANSFER hand-off edges, the accent "now" cursor, the ✕ on a failed span, the ◷ on an awaiting-human span, a dashed PLANNED ghost ahead of now, and the goldfive lane in its --hg-gf-* call-category hues (judge → --good, refine → magenta). Token-only: a theme swap re-skins every mark with no re-render. Hover any bar.

Option B — keep the canvas, restyle via tokens

harmonograf's production Gantt is a 3-layer <canvas> (background · blocks · overlay) where "React never drives the render loop" — already half the zicato digest-gating discipline. This option keeps that renderer and only changes its inputs: every fillStyle reads a resolved --hg-kind-* / --hg-gf-* / --v2-* token (via getComputedStyle) at draw time, so a theme switch re-colors on the next frame. Same trace, same encoding as Option A — every kind hue, status treatment, transfer edge and planned ghost — rendered to a real canvas here to prove the re-skin.

Trade-off. Canvas keeps harmonograf's existing hit-testing, dirty-rect batching and 60fps live cursor, but marks aren't in the DOM (no per-bar :focus-visible, no CSS hover — those are hand-rolled). The SVG option is fit-to-width and accessible by construction (every bar is role="img"-able, focusable, hoverable) but redraws the whole figure on a structural change. For the hero surface the SVG line-art is the recommendation; the canvas stays the right call for very long, deeply-zoomed live runs.
