Premise

This view answers one question: who said what to whom, and when — the agent-interaction picture, not the execution overlay. The classic answer is a sequence diagram: one lifeline per agent, time flowing down, each message a horizontal connector from sender to receiver with a clear arrowhead. We keep that grammar — it is the right one — but Tufte it into the zicato language. The rules: lifelines are identity and stay neutral (a hairline, a quiet name chip — identity never earns color); messages carry the KIND hue (transfer yellow, user green, agent-message bright-green); activations are quiet kind-hue bars on the lifeline; a delegation opens a sub-lifeline, runs beneath, and returns with its verdict; the accent appears only at now. Paired with it, the directional transfer chord summarises the same conversation as a topology — who initiates, who receives — so the time-ordered diagram and the shape of the communication agree.

The sequence diagram RECOMMENDED

One vertical lifeline per agent, time flowing down — the canonical sequence-diagram orientation, so it reads as one at a glance. Every message is a horizontal connector from the sender's lifeline to the receiver's, with a solid arrowhead at the receiving end (a synchronous call) or a dashed return (a result/reply), the label riding above in the KIND hue — dashed TRANSFER yellow for hand-offs, user green, agent-message bright-green. Activations hug each lifeline as kind-hue bars (the agent is busy). The tester delegation opens a child lifeline, runs beneath, and returns ✓. The reviewer's approval is a gate (○ deciding). The failed check is the only red. It paginates vertically like a transcript, which is exactly how an operator deep-reads a long exchange.

The transfer chord, made directional: each agent's arc splits into a solid outbound half and a dashed inbound half; every ribbon is pinched at its source and broad at its target, carries the source→target hue, and lands an arrowhead on the receiver. A coordinator-and-spokes run fans out from one sector; a web means everyone is talking to everyone.

Alternative — the score (time runs right, like the Gantt)

The same trace laid horizontally, sharing the Gantt's time axis exactly. Agents are horizontal staves; work sits ON the staff as quiet kind-hue activations; messages are thin vertical connectors with open arrowheads. Its strength is being the Gantt's connective tissue — drop it over the same axis and the two views scrub together — but as a sequence diagram it is weaker: vertical message connectors between five closely-stacked staves are cramped, and it reads as an execution overlay more than a conversation. Keep it as the Gantt-companion lane; the vertical sequence diagram above is the view for reading the interaction.

Before — the inherited frame

The textbook UML sequence diagram has the right orientation — lifelines down, messages across — but spends every mark on structure: actor boxes, a heavy lifeline grid, filled arrowheads, activation rectangles, an alt frame. None of it encodes kind, status, or cost. The recommended view keeps its grammar and drops the chartjunk.
