Premise

zicato's figure language is built on oscillation around a reference rule — good below, bad above, one accent spine. harmonograf's domain — drift and correction — is oscillation around a reference. These prototypes extend the grammar with harmonograf's nouns: the plan-rule replaces the champion-rule, the intervention replaces the gate, and the run's own trace becomes the brand. Conventions inherited whole: hairline gridlines, rx:3 bars, status earns color / identity never does, --accent only for now & selection, every figure width:100% + viewBox.

01 · the harmonograph fingerprint — a session identicon that is data

Each session gets a glyph drawn by the instrument itself: x driven by plan progress, y by execution drift, damped. Aligned runs trace tight decaying figures; detuned runs precess; a corrected run visibly tightens mid-figure where the intervention landed; a failing run spirals outward. The dot is the pen — and its color is status, earned: accent while live, good when done, bad when failed. For session pickers, fleet cards, tab titles — an identicon operators learn to read.

02 · the drift seismograph — the headline figure

Per agent: a dashed on-plan rule, a tolerance band hugging it (good-soft), and the goldfive judge series as a thin ink trace. The trace stays neutral until it escapes the band — the excursion area fills bad-soft, severity is distance, not color. Interventions are caution ticks; watch the trace decay back after each one. Answers the operator's only question — “is it wandering, did my correction take?” — with zero labels.

03 · the judge heartbeat strip — word-sized verdicts

A genDots-style strip under a lane: one glyph per judge call — · on-task, ▲ warning, ✕ critical — in the goldfive category hue at quiet opacity. The EKG of the run; trellis it across sessions for the fleet view.

04 · the intervention ladder — escalation made literal

goldfive's escalation ladder drawn as a ladder: rungs in ascending severity, the session's interventions as dots placed on their rung over time, a faint step-path connecting them. A healthy run lives on the bottom rung; a run climbing the ladder is visibly escalating. The escalate-rung dot is bad — earned, terminal.

05 · correction slopegraphs — did the steering work?

At every intervention, a two-point slope: drift before → drift after. Down-slope good, up-slope bad, flat flat — zicato's pairedSlopegraph verbatim, new noun. A row of these in the Interventions panel is an efficacy record of the operator and of goldfive's own refinements.

06 · the drift sankey — what causes what

drift kind → intervention → outcome, ribbon width ∝ count. The figure that says “tool-misuse resolves with a nudge, goal-drift recurs even after replans” — the exact shape zicato's meta-loop wants as input. Outcome nodes earn good / bad; everything upstream stays neutral ink.

07 · plan strata — revisions as sediment

Plans are geology: newest on top. Each revision is a stratum; tasks keep their column, so persistence reads as vertical alignment. A dropped task ends with ✕ at the seam; an added one enters with ○. Each dashed seam is annotated with the drift kind that forced the replan — the why is on the figure. The live stratum carries the accent spine.

08 · the replan slopegraph — one seam, magnified

Plan v2 → v3: kept tasks run flat, reordered tasks cross, dropped tasks terminate (✕), added tasks enter (○, accent-dashed). Crossing density is plan churn — ten seconds tells you whether the replan was a tweak or an upheaval.

09 · the plan reel — the spine across revisions

zicato's roundTimeline spine, one node per plan revision, annotated with remaining work — a loss-floor that should descend. Seam triggers ride above in caution; a revision that added work would step up and read as bad, earned by direction.

10 · the delegation score — fork, run beneath, rejoin

Not a UML sequence diagram — a score. A delegation branches off the parent lane, runs beneath it, and rejoins with its verdict glyph at the merge; band thickness ∝ tokens spent, so an expensive delegation is visibly heavy. Peer transfers are dashed hand-off arcs in TRANSFER yellow. elimFlow's convergence idiom, inverted into fork-and-return.

11 · the transfer chord — who talks to whom, and in which direction

Agents on an arc, transfer counts as chords, width ∝ frequency, quiet agent-ramp hues. A plain chord hides the one thing the operator most needs — who initiates vs. who receives — so this one is directional three ways at once: (1) each agent's sector is split into an outbound half (solid, source side) and an inbound half (hollow tick, target side); (2) every chord is a thin line whose hue fades from a faint source to a bright target, so it visibly flows toward whom it reaches; (3) and it ends in a small arrowhead on the target arc. A coordinator-and-spokes system fans out from one sector; if it looks like a web, something's off. The session-summary topology check.

12 · conductor's marks — steering in the margin

Operator actions live in the margin, like a conductor's pencil on the score: a magenta ◷ where the run paused for a human, an accent tick where a steering message landed, an approval gate on the lane (○ deciding → ✓ continues). The discipline: operator presence annotates the work, never recolors it — identity doesn't earn color, even human identity.

13 · the context tide — breathing room

Context-window occupancy as a tide under the lane, rising toward the dashed ceiling; the surface mixes toward caution as it nears the limit. A compaction is a relief seam — the tide visibly drops and the run breathes again.

14 · the plan DAG — interdependent processes, Adamiecki's own subject

The harmonogram's whole point in 1896 was interdependence — so the plan's dependency graph is a first-class figure, not a footnote. Tasks are quiet nodes layered by depth; dependencies are hairline edges; the critical path is the one accent spine — zicato's champion-spine idiom pointed at scheduling. Status is earned per node (✓ done, ● running on the spine, ○ pending); a task dropped at a replan stays as a dashed ghost with its severed edge — the DAG remembers the seam. Pairs with 07 strata (same tasks, time-axis vs dependency-axis).
